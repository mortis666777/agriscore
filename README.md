# AgriScore 

AgriScore — Архитектура платформы
Заявка поступает через веб-форму в браузере фермера. Он заполняет стандартные поля: область, район хозяйства, направление животноводства, наименование субсидирования, норматив и сумму. После отправки заявка получает уникальный номер и фиксируется в системе с датой поступления и статусом «На рассмотрении».
Как только заявка зарегистрирована, ИИ-модуль автоматически запускает анализ. XGBoost смотрит на комбинацию признаков: направление животноводства, норматив, регион, район и месяц подачи — и сравнивает их с историческими паттернами одобрений и отказов из реального датасета. Autoencoder параллельно проверяет, не выбивается ли заявка из нормы по совокупности параметров — например, подозрительно маленькая сумма при высоком нормативе. По итогу анализа каждой заявке присваивается скоринговый балл от 0 до 100 и формируется SHAP-разбивка: какой фактор сколько баллов добавил или отнял. Затем Claude API преобразует эти цифры в читаемый текст на русском языке — конкретную рекомендацию с объяснением, почему заявка получила именно такой балл и на что эксперту стоит обратить внимание.
Эксперт акимата открывает карточку заявки и видит рекомендацию ИИ с расшифровкой факторов. Опираясь на эту аналитику, он принимает профессиональное решение и заполняет итоговый отчёт. Решение фиксируется в системе с именем члена комиссии и обоснованием.
Итоговый отчёт формируется в следующей структуре:
№ п/п — Дата поступления — Область — Акимат — Номер заявки — Направление животноводства — Наименование субсидирования — Статус заявки — Норматив — Причитающаяся сумма — Район хозяйства — Рекомендация ИИ (балл + ключевые факторы) — Решение эксперта — Обоснование
Отчёт экспортируется в PDF и передаётся на финальное утверждение комиссии. Система не принимает решений — она даёт эксперту аналитическую основу, чтобы его вывод был прозрачным, воспроизводимым и задокументированным.
